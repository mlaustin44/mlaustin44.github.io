Getting my Desired CapsLock Behavior in Ubuntu

mlaustin
Wed Sep 02 2020
  • ubuntu

# Introduction

I use vim (or vim-mapped) editors for manipulating text and writing code. While 2020 has been an exception, I used to travel very frequently and work on a 15" Macbook Pro with a touch bar - so hitting escape was not really possible. I tried a few alternative mappings to change modes in vim and what I finally settled on was remapping my caps lock key - which is fundamentally useless for me in its normal state.

I want a very specific behavior from caps lock that works well for me:

  • A press acts as escape
  • Chording (pressing caps lock with another key(s)) acts as control

This setup does wonders for my comfort when working on a laptop keyboard since I frequently need escape and control - when remapped this way I can hit either very comfortably with my left pinky finger.

# What I Used to do

I used to use the following set of commands in my ~/.zshrc, which worked great:

setxkbmap -option 'caps:ctrl_modifier'
xcape -e 'Caps_Lock=Escape'

These map caps lock to an escape key when pressed alone and a control key when used as a modifier.

For whatever reason, these just stopped working a few weeks ago. It looks like xcape isn't being maintained any more, which led me to look for a better solution.

# The Solution - caps2esc

A quick search found that there is a utility designed to provide the exact behavior I want - caps2esc.

However, setting it up on Ubuntu is somewhat complicated - so I wanted to share my experience here.

# Installing caps2esc on Ubuntu 20.04

caps2esc depends on another set of tools designed to intercept events in linux - interception tools. We'll need to build and install interception tools and caps2esc, create a config file to intercept caps lock events, and then create a systemd service to run the interception tool and caps2esc automatically.

  1. Install prereqs

    sudo apt install -y libboost-all-dev libyaml-cpp-dev libevdev-dev libudev-dev cmake

    FYI: I install the full set of boost libraries here which are about 500mb.

  2. Clone inception tools repo, build, and install

    git clone https://gitlab.com/interception/linux/tools.git
cd tools
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
cd build
make
sudo make install

    This will build and install interception tools to /usr/local/bin

  3. Clone caps2esc, build, and install

    git clone https://gitlab.com/interception/linux/plugins/caps2esc.git
cd caps2esc
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
cd build
make
sudo make install

  4. Create a configuration for the event interception tools - this is what allows the caps lock events to be intercepted Put the following in /etc/udevmon.yaml:

    - JOB: "intercept -g $DEVNODE | caps2esc | uinput -d $DEVNODE"
  DEVICE:
    EVENTS:
      EV_KEY: [KEY_CAPSLOCK, KEY_ESC]

  5. Create a systemd service Put the following in /etc/systemd/system/udevmon.service:

    [Unit]
Description=udevmon
Wants=systemd-udev-settle.service
After=systemd-udev-settle.service

[Service]
ExecStart=/usr/bin/nice -n -20 /usr/local/bin/udevmon -c /etc/udevmon.yaml

[Install]
WantedBy=multi-user.target

  6. Start the service

    sudo systemctl enable --now udevmon
